# Finance Tracker

Live demo: https://finance-tracker-eight-lilac.vercel.app/budget

Приложение для учёта финансов: добавление доходов, распределение по категориям и работа с бюджетом.

---

## 🧰 Стек

- **Backend:** FastAPI, SQLAlchemy 2, PostgreSQL, pytest  
- **Frontend:** Vue 3, TypeScript, Vite, Tailwind CSS, FSD архитектура


---

## 🚀 AI-First подход

Проект разрабатывался с использованием Cursor и LLM как основного инструмента.

#### Основной принцип

- Человек → декомпозиция, архитектура, контроль  
- AI → генерация кода  


#### Как устроен процесс:

- Задача разбивается на небольшие изолированные части  
- Под каждую часть формируется точечный prompt  
- Полученный результат проверяется и дорабатывается через итерации с LLM  
- Итерации продолжаются до получения корректного и стабильного результата  

После этого:

- архитектурные решения выносятся в `.cursor/rules` (например, структура FSD и правила разбиения на слои)
- повторяющиеся задачи выносятся в `.cursor/skills` (например, проверка и исправление кода по заданным правилам)

#### Пример

FSD-архитектура формировалась итеративно:

- берётся одна страница  
- через серию итераций доводится до корректной структуры  
- фиксируются правила (rules)  
- добавляется skill для проверки и корректировки кода по этим правилам  

Далее при генерации нового кода:

- если правила нарушаются → даётся обратная связь LLM  
- через итерации код приводится в корректное состояние  
- при необходимости правила обновляются, чтобы избежать повторения ошибки

## 🧠 AI-инфраструктура

В проекте настроена среда, которая управляет генерацией кода через rules, skills и hooks.


### Rules

- `fsd-structure.mdc` — фиксирует FSD-структуру фронтенда: слои, зависимости, правила размещения API, UI и логики  
- `api-contracts.mdc` — описывает backend API, формат ошибок, эндпоинты и требования к тестам  
- `backend-conventions.mdc` — задаёт соглашения для FastAPI / SQLAlchemy / Pydantic v2 и формат ошибок через `AppError`  
- `create-page.mdc` — описывает создание страницы: структура файлов, роутинг и соответствие FSD  


### Skills

- `fsd-review` — проверяет frontend-код на соответствие FSD и исправляет нарушения  
- `test-backend` — запускает backend-тесты и помогает устранить выявленные ошибки  


### Hooks

- `lint-on-edit.sh` — после изменения файлов:
  - для `.ts` / `.vue` — применяет ESLint + Prettier  
  - для `.py` — применяет Ruff
